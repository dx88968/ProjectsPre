Sharemi Project

Section 1: What is Sharemi?

Sharemi is a implementation of location based file service.
Users can upload local files to our file server using a local App we provide, and after that he can:

  • Enable any other Sharemi who is near enough to him can download the file.
  • Other user who bump his phone with him can download the file.

Here are some snaps of this App.

After logged in, user can pick a file to share by pressing "Share Files".

Picture 1-3:Pick a file to share

When a file has been shared, Other user who are within 10 metres from the file owner will see this file in "Shared with me".

Section 2: Sharemi Architecture

Picture 2-1:Architecture of Sharemi

Sharemi client is fundmentally a web broswer with some plugins accessing local devices. And the server is actually a PHP website which records all online users' current GPS location and his shared files.

Section 3: Future of Sharemi

Feature 1: bond location with service

Sharemi in the future can not only register files with the customer's current location to share with the other customers around him, but also be capable to bond files/service with a specified location. Like in the picture 4-1, the manager of Barco Law Building registers some files with the law building in the Sharemi.

Picture 4-1

In this case, when a Sharemi customer enters the area of Barco Law Building, the shared files of Barco Law Building will appear on the Sharemi (Like Picture 4-2).

Picture 4-2

Feature 2: Implementation of Ubiquitous Computing

Customers can register all kinds of service in Sharemi using Sharemi-locationService-Bridge. For example, if there is a printer connected to a computer, then someone can use Sharemi-locationService-Bridge to register the printer in Sharemi, which enables people near the printer to use Sharemi to print files with that printer.
For example, manager of Barco Law Building want to register a public printer for students who near the building, so he uses Sharemi-locationService-Bridge (Picture 4-3).

Picture 4-3

After he clicks "Enable Bridge", students near Barco Law Building will see this printer service in Sharemi (Picture 4-4).

Picture 4-4

Besides print service, there are other kinds of local service can be shared by Sharemi like:

  • local Web Server (Individual homepages)
  • local Web Service
  • ...

The greatest technology breakthrough in Sharemi Ubiquitous Computing is that this idea can be implemented without additional devices/sensors, which makes it easy to promote to the public.

Estate Bubble Simulator Project

In order to generate convincing results, I have to implement a system which allows hundreds of independent agents running in the same time, these agents perform the people of different roles in the real estate market, they make decisions based on BDI(Brief-Desire-Intention) model(means the agents will always do thing they most want to do). This is really big project , so I divide it into 4 modules:

  • Buyer/Saler Agents
  • Big Environment where Agents live
  • Special Module of monitoring agents and controlling the system
  • GUI module which tells the users what is going on in the system

ZooKeeper Project

A java swing based graphics PC game which supports LAN battles.
Game Rule: though swapping Adjacent animals to get three or more same animals connected to each other, the player will get these animals removed and get some points. And the person who reach the reach the required score in a limited time(Stort mode) or ahead of his opponent(LAN Mode) will win.

This project can also be divided into 4 modules: GUI Support, Game Logic, Network, File Management.

As a graphics game, the basic UI function which provided by java swing is far from enough, so I designed my own UI libaray extension to java swing.Click to see the class diagram of GUI module

Also, this game is running on a complex game logic.Click to see the class diagram of Game Logic module
